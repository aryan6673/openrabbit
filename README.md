## Overview

OpenRabbit is a free (you can even get a free llm api explained below), open-source, self-hosted GitHub Pull Request reviewer. It analyzes PR diffs, consults a pluggable LLM provider (Groq / OpenRouter / others), and posts a concise, structured review: a human-readable summary and accurate inline comments or suggestions.

OpenRabbit emphasizes accuracy and safety. It enforces a strict JSON response schema, validates inline suggestion relevance, and maps diffs to GitHub review positions using content-aware heuristics.

## Quick Start

1. Add the remote workflow to the repo you want to review (recommended — runs the action from `main` safely using `pull_request_target`):

```yaml
name: OpenRabbit
on:
  pull_request_target:
    types: [opened, reopened, edited, synchronize]

permissions:
  contents: read
  pull-requests: write

jobs:
  review:
    runs-on: ubuntu-latest
    steps:
      - name: OpenRabbit
        uses: aryan6673/openrabbit@main
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          # Optional: post as a machine user (PAT or installation token)
          bot_token: ${{ secrets.REVIEW_BOT_TOKEN }}
          llm_provider: openrouter
          llm_api_url: https://openrouter.ai/api/v1
          llm_api_key: ${{ secrets.LLM_API_KEY }}
          llm_model: openrouter/free
          review_mode: both
          tone_mode: balanced
```

2. Or run locally via the CLI:

```bash
git clone https://github.com/aryan6673/openrabbit.git
cd openrabbit
npm install
npm run build
node dist/cli.js --owner=ORG --repo=REPO --pull-number=123 \
  --github-token="$GITHUB_TOKEN" --llm-api-key="$LLM_API_KEY" \
  --bot-token="$BOT_TOKEN" # optional
```

3. Get an LLM API key from your provider (Groq/OpenRouter or another OpenRouter-compatible LLM).

## Features

- **Human-quality summary + inline comments:** high-level overview plus focused, actionable inline suggestions.
- **Structured JSON output:** the LLM is prompted to return a strict JSON schema so reviews are machine-readable and auditable.
- **Two-stage file fetch:** the model can request additional files; OpenRabbit fetches only what the model needs to reduce token usage.
- **Content-aware mapping:** maps line numbers in diffs to GitHub review `position` with heuristics to avoid mis-placed comments.
- **Suggestion validation:** suggestions are checked against file content before posting inline; invalid suggestions are appended to the review body instead.
- **Dependabot-aware behavior:** tiny or autogenerated PRs get concise, lockfile-focused reviews.
- **Pluggable LLM providers:** built-in adapters for Groq and OpenRouter; add your own provider via the `src/llm` adapter interface.
- **Optional bot user:** post reviews as a machine user by supplying a `bot_token` (PAT or installation token).

## Review Modes

- **summary:** single summary review comment (no inline comments).  
- **inline:** post only inline comments and suggestions.  
- **both:** post both the summary and inline comments (default).

## Privacy

OpenRabbit does not run a central server. The only data leaving your environment are API calls to the configured LLM/transcription provider(s). Keep your `LLM_API_KEY` and optional `bot_token` in repository or organization secrets and restrict access as appropriate.

## Prompting & Customization

You can configure the system prompt used for reviews. OpenRabbit expects the model to return structured JSON. Here's a minimal example of the output schema OpenRabbit enforces:

```json
{
  "summary": {
    "overview": "Short human-readable summary",
    "verdict": "accept|minor|major|decline",
    "primaryGoal": "What the PR aims to accomplish",
    "scopeAssessment": "Was scope respected?",
    "riskAssessment": "High/medium/low plus brief rationale",
    "reuseNotes": ["notes"],
    "actionItems": ["tasks"]
  },
  "comments": [
    {
      "path": "src/file.ts",
      "line": 42,
      "body": "What needs to change",
      "type": "bug",
      "suggestion": "replacement code snippet (optional)"
    }
  ],
  "requested_files": ["path/to/other/file.ts"]
}
```

If you prefer a simpler system prompt or different JSON shape, change the prompt in `src/reviewer.ts` and adapt any downstream integrations.

## FAQ

**Why use Groq/OpenRouter instead of a local model?**  
Running a local LLM for high-quality post-processing increases latency and resource usage. Using a hosted LLM gives a faster, smoother UX for conversational-style reviews. You can still self-host OpenRabbit and keep your API keys in org secrets.

**How do I post comments from a bot account?**  
Supply a `bot_token` (PAT or GitHub App installation token) via `bot_token` input or `BOT_TOKEN` env var. OpenRabbit will use the bot token for API calls when present.

**Is it safe to run as `pull_request_target`?**  
Yes — `pull_request_target` runs actions in the context of the base branch and is the recommended way to use remote actions that need secrets. Limit which repos can access the org secret for additional security.
